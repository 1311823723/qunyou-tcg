# UI Style Guide

## 视觉关键词

- 暗色桌面
- TCG 卡牌感
- 群友整活
- 清楚易读
- 轻微霓虹
- 不要廉价页游风
- 不要后台管理系统风

## 颜色方向

### 基础背景

- 深黑
- 深蓝
- 深紫
- 石墨灰

### 流派强调色

| 流派 | 强调色 |
|---|---|
| 爆杀流 | 红、橙、金 |
| 锦囊流 | 蓝、紫 |
| 制衡流 | 青、绿 |
| 控制流 | 紫、黑、暗红 |

## 卡牌组件

卡牌应该像"收藏册里的卡"，不是普通表格。

每张卡至少包含：

- 清晰标题
- 类型 / 定位标签
- 费用
- 发动时机
- 技能名
- 技能效果

技能效果文本必须有足够行高。不要把技能字压得太小。不要用纯图片承载文字。

## 标签样式

标签使用小胶囊样式。

**主定位标签更突出：** 强攻、防御、资源、控制、支援、伏击

**机制标签稍弱：** 爆发、连杀、退场、检索、破防、卖血、响应、回收

## 页面布局

### 首页

- 要有强视觉开场
- 要让人 5 秒内知道这是个什么游戏
- 要能直接进入规则、卡牌、预组

### 卡牌页

- 网格布局
- 支持筛选
- 支持搜索
- 移动端单列
- 桌面端多列

### 预组页

- 顶部展示预组风格
- 中间展示本体和 Mega
- 下方展示角色牌
- 侧边或下方展示标签统计和打法说明

## 禁止事项

- 不要把卡牌做成纯表格
- 不要大段白底黑字
- 不要让技能文本溢出卡牌
- 不要为了炫技加入影响阅读的动画
- 不要在 UI 里硬编码卡牌数据
- 不要擅自修改卡牌效果
