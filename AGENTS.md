# AGENTS.md

本文件为 AI coding agent 提供项目约定。请遵守以下原则：

1. 所有正式卡牌数据必须写在 `data/cards/*.json`。
2. 所有预组列表必须写在 `data/decks/*.json`。
3. 不要在 UI 里硬编码卡牌效果，UI 应读取 JSON 数据。
4. 修改卡牌效果后，必须同步更新 `playtest/changelog.md`。
5. 修改规则后，必须同步更新 `docs/rules.md` 和 `docs/keywords.md`。
6. 新增角色牌时必须包含：
   - id
   - name
   - cardType
   - deck
   - mainRole（取值：强攻/防御/资源/控制/支援/伏击）
   - tags
   - cost（格式：`{ "type": "休整", "amount": 1 }` 或 `{ "type": "退场", "text": "退场自身" }`）
   - timing
   - skillName
   - effectText
7. 每次改动尽量小，不要一次性重写整个项目。
8. 如果不确定规则，不要擅自发明大机制，先在 `playtest/balance-notes.md` 里记录建议。
9. UI 开发前必须阅读 `docs/style-guide.md`，遵循暗色桌面、TCG 卡牌感、轻微霓虹的视觉方向。
10. 卡牌展示不能做成表格，必须像"收藏册里的卡"。标签使用小胶囊样式。
11. 技能效果文本必须有足够行高，不能压得太小，不能溢出卡牌。
12. **角色名分隔符必须使用 `-`（连字符），禁止使用 `·`（间隔号）。**
    例：`刺客-柯柯` ✅ / `刺客·柯柯` ❌
    原因：前端 `splitCharacterName()` 用 `indexOf("-")` 分割职业与群友名以实现两行显示。
13. **ID 命名前缀规则：**
    - 本体：`body_[流派]_[序号]`（如 `body_aggro_001`）
    - 角色：`char_[全局序号]_[群友英文]_[身份英文]`（如 `char_001_keke_assassin`）
    - 角色编号从当前角色牌文件顺序全局递增，不随预组或流派重置
    - 角色身份使用英文意译，群友使用稳定英文或拼音 slug，均为小写，可用连字符
    - 预组：`deck_[流派]_[序号]`（如 `deck_trans_001`）
    - 预组页面 slug：`getDeckSlug()` 自动从 ID 提取中间部分
14. **角色复用规则：**
    - 同一张角色可被多个预组引用（跨预组引用允许）
    - 同一预组内的每张角色技能必须唯一
    - 优先使用现有角色，不足时再新增角色牌
15. **现有预组一览：**

    | 流派 | 预组名 | 本体 | 核心玩法 |
    |------|--------|------|----------|
    | 爆杀流 | 上头组 | 微笑尅乐 | 找【杀】→强化→压血 |
    | 密裁 | 密裁组 | 柯柯 | 看情报→控资源→致命清算 |
    | 锦囊流 | 操作组 | 瓜猫 | 用锦囊→摸牌→回收→循环 |
    | 拟态流 | 变通组 | 摆子 | 转化→视为打出→复制→回收→Mega后弃牌区任意用 |

16. **本体设计规则：** 体力固定 7，技能是流派发动机提供牌差收益。Mega 条件是技能强化版，通过核心操作触发。
17. **工作流：** 修改卡牌数据后 → 运行 `npm run validate`；修改前端后 → 运行 `npm run build`。新增预组时需：新建 `.deck.json`、在 `src/lib/decks.ts` 注册、创建 `.astro` 页面。
